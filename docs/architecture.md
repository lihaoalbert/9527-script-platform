# 技术架构

## 架构策略

MVP 采用模块化单体：

- 前端和后端分应用管理。
- 后端按业务模块划分，未来可拆为独立服务。
- 数据一致性优先，剧本独占锁定以 PostgreSQL 事务为准。
- Redis 用于缓存、限流和异步任务，不作为版权状态最终来源。

## 服务边界

```text
apps/web
├─ 剧本库
├─ AI创作工作台
├─ 剧本评分
├─ 我的积分
└─ 管理后台

apps/api
├─ AuthModule
├─ ScriptsModule
├─ CreditsModule
└─ AiModule
```

## 剧本锁定流程

```text
检查剧本状态
-> 检查积分账户
-> 扣减积分
-> 写入积分流水
-> 创建锁定记录
-> 更新剧本状态为 LOCKED
```

所有步骤在同一个数据库事务内完成。

## 后续拆分方向

- AI 生成服务：长文本任务、模型调度、token 成本追踪。
- 搜索服务：中文分词、标签搜索、语义搜索。
- 文件导出服务：txt/md/docx/pdf。
- 推荐服务：相似剧本、爆款标签、题材趋势。
